# AAEngine Agent 配置说明

## Agent 概述

本项目配置了5个专业的AI Agent，采用流水线工作模式进行软件开发。

## Agent 配置

### 1. 产品规划师 (product-planner)
- **角色**: 负责产品规划和项目管理
- **职责**:
  - 需求分析和文档编写
  - 产品功能规划
  - 任务分解和优先级排序
  - 项目进度跟踪
  - 团队协调和沟通

### 2. 架构设计师 (architect)
- **角色**: 负责架构设计
- **职责**:
  - 系统架构设计
  - 技术栈选型
  - 数据库设计
  - API接口设计
  - 模块划分和依赖管理

### 3. 开发工程师1 (developer-1)
- **角色**: 负责代码实现
- **职责**:
  - 核心功能开发
  - 代码编写
  - 单元测试
  - Bug修复

### 4. 开发工程师2 (developer-2)
- **角色**: 负责代码实现
- **职责**:
  - 辅助功能开发
  - 代码编写
  - - 单元测试
  - Bug修复

### 5. QA工程师 (qa-engineer)
- **角色**: 负责代码review和测试
- **职责**:
  - 代码审查
  - 测试用例编写
  - 集成测试
  - 质量保证
  - **测试通过后自动提交代码到GitHub**

## 工作流程

### 阶段1: 规划阶段
- 执行Agent: 产品规划师
- 输出: 产品需求文档

### 阶段2: 设计阶段
- 执行Agent: 架构设计师
- 输入: 产品需求文档
- 输出: 架构设计文档

### 阶段3: 开发阶段
- 执行Agent: 开发工程师1, 开发工程师2 (并行)
- 输入: 架构设计文档
- 输出: 实现代码

### 阶段4: 测试阶段
- 执行Agent: QA工程师
- 输入: 实现代码
- 输出: 测试报告和代码提交到GitHub

## 使用方法

### 方式1: 使用工作流管理器
```bash
python3 .agents/workflow.py
```

### 方式2: 手动调用各Agent
根据项目需求，可以手动调用对应的Agent执行特定任务。

## Git 工作流

1. 产品规划师创建需求文档
2. 架构设计师创建架构文档
3. 开发工程师实现功能
4. QA工程师进行代码审查和测试
5. 测试通过后，QA工程师自动提交代码到 `https://github.com/chaov/aaengine.git`

## 配置文件说明

### config.json
- 定义了所有Agent的配置信息
- 定义了工作流的各个阶段
- 定义了Agent之间的依赖关系

### workflow.py
- Python脚本，用于管理工作流
- 支持运行完整工作流或单个阶段
- 支持提交代码到GitHub
